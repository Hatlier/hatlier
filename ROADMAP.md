# Hatlier ロードマップ & 見える化

開発の「いま・次・あと」をここに置く。  
**リリースノート（過去の変更）より、予定と優先度の方が先** — この文書を正本にする。

画面左上の `vX.Y.Z` はこの ROADMAP へリンクする。

---

## 運用の考え方

| 置き場 | 役割 |
|---|---|
| **`ROADMAP.md`（本ファイル）** | これからやること・やらないこと・段取り。開発と直結 |
| **`SPEC.md`** | 思想・原則・詳細仕様 |
| **GitHub Releases** | 区切りのよい版だけタグを打つ（任意）。毎回の push には付けない |
| **Commit / PR** | 何を変えたかの履歴。細かいリリースノート代わり |

**やらない運用:** 毎 push にリリースノートを義務づける（しんどい・形骸化する）。

**やる運用:**

1. 機能を入れる前に、本 ROADMAP の「Next」へ一行足す（または移動する）
2. 入れ終わったら「Done」へ移し、`HATLIER_VERSION` を上げる
3. 大きな区切り（例: 0.3 → 0.4）のときだけ GitHub Release を切る
